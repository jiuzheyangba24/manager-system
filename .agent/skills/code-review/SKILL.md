# 代码审查技能

本技能用于对代码进行系统性审查，确保代码质量和安全性。

## 审查维度

### 1. 正确性 (Correctness)
- 代码是否实现了预期功能
- 边界条件是否处理正确
- 错误处理是否完善

### 2. 安全性 (Security)
- 是否存在 XSS 漏洞
- 是否正确验证用户输入
- 敏感数据是否加密
- API 调用是否安全

### 3. 性能 (Performance)
- 是否有不必要的重复计算
- 是否有内存泄漏风险
- 是否优化了大数据处理
- 是否使用了适当的缓存策略

### 4. 可维护性 (Maintainability)
- 代码是否易于理解
- 命名是否清晰有意义
- 是否有适当的注释
- 函数是否足够简洁

### 5. 代码风格 (Style)
- 是否遵循项目代码规范
- 缩进和格式是否一致
- 是否有未使用的变量或导入

## 审查清单

### Vue.js 组件审查
- [ ] Props 是否有类型定义
- [ ] 是否正确使用了 ref 和 reactive
- [ ] computed 是否有副作用
- [ ] watch 是否正确清理
- [ ] 模板中是否有复杂表达式

### JavaScript 审查
- [ ] 是否使用了严格等于 (===)
- [ ] 异步操作是否有错误处理
- [ ] 是否避免了全局变量
- [ ] 是否正确处理了 null/undefined

### CSS/TailwindCSS 审查
- [ ] 是否有重复的样式定义
- [ ] 响应式设计是否完善
- [ ] 是否避免了 !important
- [ ] 动画是否流畅

## 审查反馈格式

```markdown
## 代码审查结果

### ✅ 优点
- 列出代码的优点

### ⚠️ 建议改进
- 列出建议改进的地方

### 🚫 需要修复
- 列出必须修复的问题

### 📝 总结
简要总结审查结果
```

## 严重性级别

| 级别 | 说明 | 行动 |
|------|------|------|
| 🔴 Critical | 安全漏洞或严重bug | 必须立即修复 |
| 🟠 Major | 功能问题或性能问题 | 应该修复 |
| 🟡 Minor | 代码质量问题 | 建议修复 |
| 🟢 Info | 风格建议 | 可选修复 |
